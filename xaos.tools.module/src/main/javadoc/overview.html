XAOS (ΧΑΟΣ – the primordial state of existence) is a JavaFX application framework.

The framework was initially conceived to create accelerator physics applications based on Open XAL, an open source development environment used for creating accelerator physics applications, scripts and services.

The framework is composed by a series of modules: the XAOS Tools module provides tools used by the other modules (e.g. annotation processors).

The project is hosted in the XAOS GitHub repository.

Tools and Utilities

Service Providers

The {@link se.europeanspallationsource.xaos.tools.annotation.ServiceProvider} annotation will simplify providing service implementations, taking care of publishing them in the proper file inside the META-INF/service folder. 

  package my.module;
  import my.module.spi.SomeService;
  import se.europeanspallationsource.xaos.tools.annotation.ServiceProvider;

  @ServiceProvider(service=SomeService.class)
  public class MyService implements SomeService {
    ...
  }

Moreover, the {@link se.europeanspallationsource.xaos.tools.annotation.ServiceLoaderUtilities} class will complement the {@link java.util.ServiceLoader} one with few more methods.

Note: when using {@link se.europeanspallationsource.xaos.tools.annotation.ServiceProvider} or {@link se.europeanspallationsource.xaos.tools.annotation.ServiceProviders} the {@code service} class must be listed in the {@code module-info} class inside a uses statement. Moreover a provides … with statement must also be added to declare the annotated class as provider for the parameter class.

Note: when using {@link se.europeanspallationsource.xaos.tools.annotation.ServiceLoaderUtilities} the service provider interface type must be listed in the {@code module-info} class inside a uses statement.

Java Language Tools

The {@link se.europeanspallationsource.xaos.tools.lang.Reflections} class provides few methods to handling fields and method access using the Java Reflection API.

Note: when using {@link se.europeanspallationsource.xaos.tools.lang.Reflections} it is necessary to open the module/package being reflected to xaos.tools. This can be achieved with the following flag being added to the command line launching the application: 

  --add-opens module/package=xaos.tools
